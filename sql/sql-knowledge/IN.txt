IN
1. Concept Explanation

What it is: Operator to check if a value matches any value in a list.

Why it is needed: Cleaner alternative to multiple OR conditions.

Where it is used: Filtering by multiple values, subquery results, category filters.

2. Syntax (MSSQL)

sql
SELECT columns
FROM table
WHERE column IN (value1, value2, ...);

-- With subquery
WHERE column IN (SELECT column FROM table2);
3. Simple Example

sql
-- List of values
SELECT * FROM Customers
WHERE Country IN ('USA', 'Canada', 'UK', 'Australia');

-- With NOT
SELECT * FROM Products
WHERE CategoryID NOT IN (5, 7, 9);

-- Date list
SELECT * FROM Orders
WHERE DATEPART(MONTH, OrderDate) IN (1, 2, 3); -- Q1 months
4. Practical / Interview Example
Problem: Find all products that have never been ordered, but only in specific categories.

Solution:

sql
-- Using NOT IN with subquery
SELECT 
    p.ProductID,
    p.ProductName,
    p.CategoryID,
    p.Price
FROM Products p
WHERE p.CategoryID IN (1, 3, 5) -- Specific categories
    AND p.ProductID NOT IN (
        SELECT DISTINCT ProductID 
        FROM OrderDetails
    )
ORDER BY p.CategoryID, p.ProductName;
5. Important Notes / Common Mistakes

IN is faster than multiple ORs for same column

NULL values in IN list are ignored

NOT IN with NULL in subquery returns no rows (use NOT EXISTS instead)

For long lists, consider temp table or table variable

Common mistake: Using IN for very large lists (performance issue)
